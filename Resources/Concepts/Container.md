# Container

A lightweight, standalone, executable package of software that includes everything needed to run an application: code, runtime, system tools, system libraries and settings.

> From: [[Processed_Transcripts/Kubernetes Overview.md]] (Added on 2026-03-01 16:42)

*   Containers are a good way to bundle and run your applications.

*   Similar to a VM, a container has its own filesystem, share of CPU, memory, process space, and more. As they are decoupled from the underlying infrastructure, they are portable across clouds and OS distributions.
