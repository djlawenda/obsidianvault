# Traditional Deployment

An early method of deploying applications directly on physical servers, often leading to resource allocation issues and underutilization.

## Excerpts & Key Information


> From: [[Processed_Transcripts/Kubernetes Overview.md]] (Added on 2026-03-01 16:42)

*   Traditional deployment era: Early on, organizations ran applications on physical servers. There was no way to define resource boundaries for applications in a physical server, and this caused resource allocation issues.

*   A solution for this would be to run each application on a different physical server. But this did not scale as resources were underutilized, and it was expensive for organizations to maintain many physical servers.
