# Node Controller

A type of Kubernetes controller responsible for noticing and responding when nodes go down, and for checking with the cloud provider if a node has been deleted.

## Excerpts & Key Information


> From: [[Processed_Transcripts/Cluster Architecture.md]] (Added on 2026-03-01 16:42)

*   Node controller: Responsible for noticing and responding when nodes go down.

*   Node controller: For checking the cloud provider to determine if a node has been deleted in the cloud after it stops responding
